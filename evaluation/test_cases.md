# Health Pilot 完整评测用例

## 1. 评测范围

- Coach 对话主流程
- 用户档案管理与目标设定
- 饮食记录（文本/图片）与营养估算
- 体重记录与趋势判断
- 运动记录与热量消耗估算
- 数据分析（日/周/月摘要）
- 定时与条件推送
- 通知 REST API
- 基础稳定性与异常处理

## 2. 评测环境

- Python: 3.13+
- 依赖管理: uv
- 数据库: SQLite（默认）
- 启动命令:

```bash
uv sync
uv run python main.py
```

- 服务端口: 7777

## 3. 评测账号与数据前置

- 默认测试用户 ID: `test_user_001`
- 初始档案:
  - 性别: 男
  - 年龄: 28
  - 身高: 175cm
  - 初始体重: 80kg
  - 目标体重: 70kg
  - 活动水平: 久坐
- 推送时间:
  - 早餐提醒: 07:30
  - 午餐提醒: 11:30
  - 晚餐提醒: 17:30
  - 称重提醒: 08:00
  - 周报: Sunday 20:00

## 5. 异常与边界用例

| 用例ID | 场景 | 输入 | 预期结果 |
|---|---|---|---|
| EC-001 | 空输入 | 空文本 | Agent 友好提示并引导补充信息 |
| EC-002 | 非法体重 | “体重-5kg” | 拒绝写入并提示输入不合法 |
| EC-003 | 极端饮食量 | “一顿吃了20份炸鸡” | 给出异常确认，避免直接污染数据 |
| EC-004 | 时间冲突 | 设置无效时间格式 | 校验失败并返回可修正提示 |
| EC-005 | 数据库不可用 | 模拟 DB 连接失败 | 返回可观测错误，服务不崩溃 |
